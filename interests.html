What I’m Learning

Over the past few months I’ve been learning how to design clear visualizations using HTML, CSS, and JavaScript. I started with basic static charts and gradually moved toward D3 so I can build interactive explanations. I’m especially interested in how small design details—labels, spacing, color contrast, and annotation—can make a chart feel intuitive instead of overwhelming. My process usually starts with a question, then a quick sketch, and finally a prototype in the browser. From there, I test the interaction: does the reader know where to look? Is the legend placed where it helps, not where it distracts? Is the y-axis scale appropriate for the message? When I run into problems, I try to reduce the design to its essentials and build back up. This approach keeps the code organized and helps me explain the trade-offs I’m making. I also care about accessibility: using semantic HTML, sufficient color contrast, keyboard focus, and descriptive alt text. These aren’t afterthoughts; they shape how I structure the page. My goal is to create visuals that are both useful and pleasant—something I’d be proud to share in a portfolio or class presentation.
